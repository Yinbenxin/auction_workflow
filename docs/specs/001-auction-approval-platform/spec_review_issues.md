本轮复查后，上一轮保留的 4 个问题均已修复。当前未发现需要继续保留在问题清单中的阻塞性或一致性问题。

## 本轮已确认修复的问题

### 1. `US-09` 风险等级表述已修复

**所在章节：** `用户故事` → `阶段 04：策略评审确认` → `US-09 提交策略版本待确认`

**原问题：**

`US-09` 之前写“系统自动判断风险等级”，与 `业务规则` → `策略风险等级` 中 `risk_level` 由策略负责人手动维护、确认人集合以红线字段检测结果为准的规则存在口径冲突。

**当前状态：已修复**

当前 `US-09` 已改为：

```text
系统自动检测是否涉及策略红线字段变更，并据此判断所需确认人集合
```

该表述已与 `业务规则` → `策略风险等级` 的口径一致。

### 2. 阶段 10 归档门控条件已修复

**所在章节：** `业务规则` → `竞拍流程阶段状态机`

**原问题：**

`阶段10 归档` 门控条件之前使用“未确认应急修改已补说明”的表述，未完整覆盖正常临场修改闭环、预授权应急、非预授权应急等状态。

**当前状态：已修复**

当前门控条件已改为：

```text
关联最终版本、复盘必填项完整、发现问题已创建整改项、临场修改记录已闭合、应急执行已补说明或已标记流程偏差
```

并补充说明了“临场修改记录已闭合”的含义，与 `Edge Cases` → `EC-5` 和临场修改状态机口径一致。

### 3. 临场修改留痕率统计口径已修复

**所在章节：** `业务规则` → `管理指标统计口径`

**原问题：**

`临场修改留痕率` 之前按“有完整审批记录的修改数 / 总修改数”计算，无法覆盖预授权应急和非预授权应急执行。

**当前状态：已修复**

当前指标已改为：

```text
临场修改留痕率 = 有完整管控记录的修改数 / 总修改数
```

并明确“完整管控记录”包含：

- 正常修改：申请、审批、复核、执行记录齐全；
- 预授权应急：命中预授权规则、执行记录、事后补说明齐全；
- 非预授权应急：执行记录、流程偏差标记、偏差原因说明齐全。

该口径已能覆盖应急执行场景。

### 4. 执行前复核清单覆盖范围已修复

**所在章节：** `业务规则` → `双人复核规则`

**原问题：**

执行前复核清单之前未明确覆盖补量策略、任务顺序和任务启停状态。

**当前状态：已修复**

当前复核清单已从 12 项调整为 13 项，并新增/合并覆盖：

- 垫子/补量策略是否配置正确；
- 任务顺序和启停状态是否正确。

同时，`US-15`、阶段推进门控和核心验收标准中的复核项数量也已同步为 13 项。
